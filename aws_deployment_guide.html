4. EC2 Server Setup

All application processes run on a single EC2 t3a.small instance managed by Supervisor:

EC2 t3a.small (2 vCPU, 2 GB RAM) ├── Nginx → Reverse proxy, SSL termination, static file serving ├── Gunicorn → Django WSGI server (4 workers) ├── Celery Worker → Background tasks: report generation, emails, file ops ├── Celery Beat → Scheduled tasks: cron jobs, report scheduling └── Supervisor → Process manager — auto-restarts crashed processes
Gunicorn Configuration
# gunicorn.conf.py workers = 4 # 2 × vCPUs (standard rule) worker_class = "sync" timeout = 120 keepalive = 5 max_requests = 1000 # Restart workers periodically (prevent memory leaks) max_requests_jitter = 100 bind = "0.0.0.0:8000" accesslog = "/var/log/gunicorn/access.log" errorlog = "/var/log/gunicorn/error.log"
Storage
  • Use EBS gp3 volume (30 GB) — faster than gp2 at same cost (~$2.40/month)
  • Enable EC2 Auto Recovery (free) — automatically restarts the instance on underlying hardware failure
  • Assign an Elastic IP (free while attached) — static IP that survives reboots

7. S3 + CloudFront Setup

S3 stores all media files, static assets, and generated reports (PDF/Excel). CloudFront serves them globally at low latency, reducing load on your EC2.

11. Deployment Process (Zero Risk)
  1. Take a manual RDS snapshot before every deployment (takes ~2 minutes, free)
  2. SSH into EC2, pull latest code from git repository
  3. Install dependencies: pip install -r requirements.txt
  4. Run migrations: python manage.py migrate --noinput
  5. Collect static files: python manage.py collectstatic --noinput
  6. Reload Gunicorn gracefully: sudo supervisorctl restart gunicorn (existing connections finish before restart)
  7. Reload Celery workers: sudo supervisorctl restart celery
  8. Verify health endpoint responds: curl https://yourdomain.com/api/health/
ℹ Minimizing Downtime During Deploys
Use Gunicorn's --graceful-timeout (default 30s) — it lets active requests finish before restarting workers. Total downtime is effectively 0–5 seconds. Schedule deployments during low-traffic hours for safety.

⚠ Key Limitations to Communicate to Stakeholders
At $40/month, this architecture handles hundreds of daily active users and thousands of total registered users comfortably. However, it has a single app server and single-AZ database — meaning a hardware failure or AZ outage could cause 5–15 minutes of downtime. For guaranteed 99.9% uptime SLA, budget $75–110/month.
